逻辑、移位操做与空指令的实现

发表于 2019-09-08 | | 阅读次数:
字数统计: 766

概述

之前学习了OpenMIPS五级流水线结构，实现了一条ori指令，接着学习逻辑、移位操做与空指令的实现。

了解数据相关的问题

由于处理器在实现指令的过程中，本质上就是对寄存器中的值进行读取、判断、回写。而在众多指令间工作的也就只有32个寄存器加几个特殊用处的寄存器罢了。所以在寄存器的读取和写入是必须要注意的问题，一个寄存器同时进行读取和写入的操做，边吃边吐骨头可是很难的。

其实这一点在ori指令实现时的regfile模块中已经接触过，即如果要读的寄存器地址和要写入的寄存器地址相同，就直接将要写入的数据作为结果输出，而不经过写入操做。如此就解决了相隔两条指令存在数据相关的情况。

采用数据前推的方法解决流水线数据相关问题

也就是说，将执行阶段和访存阶段的指令的运算结果向前推至译码阶段，因为流水线在执行阶段就完成了指令的操做。

测试数据相关问题解决效果

通过龙芯提供的交叉编译工具，写出四条指令，分别是34011100、 34210020、
34214400、
34210044

按此逻辑结构，第一条指令回写至$1寄存器的时候已经进行到访存阶段，而此时第二条指令、第三条指令因该在译码阶段获取$1的值，也就是第一条指令的执行和访存阶段。而指令可以在执行阶段就将结果算出，即通过此途径，对第二条第三条指令进行判断，如果有指令进程过程中有寄存器的存取问题，即直接从第一条指令的执行阶段和访存阶段将算得的数据发出到对应需要的地方。

移动操做指令的实现

发表于 2019-09-08 | | 阅读次数:

概述

之前学习了OpenMips的ori指令，及其相关的逻辑指令的实现，接着将实现移动指令操做

移动操做指令说明

移动操做指令6条：movn、movz、mfhi、mthi、mfho、mtlo，后四条指令涉及对特殊寄存器HI、LO的读/写操做，这两个寄存器用于保存乘法、除法的结果。当保存乘法结果时，HI保存结果的高32位，LO保存低32位；当用于保存除法结果时，HI用于保存余数，LO负责保存商。

新的数据相关的问题

mfhi和mflo指令的处理过程需要读取HI、LO寄存器的值，所以对这两个寄存器的读取则需要注意。

这个咋一看差点忘了原理，总觉得第4条指令的调用就应该受到影响，其实不然，数据相关的关键点是对同一个寄存器的读取产生矛盾点才会产生数据相关的问题，而3、4、5这三条指令都只是写入HI寄存器而已，知道第6条指令才又开始读取HI寄存器，这个时候才会产生数据相关。并不是一开始想的不停写入只影响第6条指令。逻辑重点也理清了。

系统结构修改

由于又有两个新寄存器HI、LO的参与，在写模块时当然需要增加他们对应的模块。

避免数据相关问题，也就意味着务必要对这两个寄存器的读取进行判断，写使能，输出输入，时钟复位信号。

HILO模块

逻辑很简单，如果复位就全置为0，否则在写使能的情况下将输入赋给输出即可。

修改ID模块

修改EX模块

因为执行阶段的时候，后四条指令会用到HILO寄存器，所以在EX模块一顶会对HILO的输入输出进行再次详细的关联。

其他模块的修改
